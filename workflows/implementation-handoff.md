# Implementation Handoff

Use this workflow when a report action becomes an implementation task for a website repo, WordPress site, developer, or client team.

## Handoff fields

- Action title
- Source report/month
- Target URL or asset
- Current state
- Desired state
- Acceptance criteria
- Owner: Us / Client / Developer / Client + Developer
- Expected SEO signal and timeframe
- Rollback or risk notes

## Rules

- Do not make customer-facing website changes without approval.
- Verify rendered output after implementation, not just source files.
- Keep implementation tasks small enough to complete before the next reporting cycle.
